Manual Configuration Guide

Anchorr is a self-hosted Discord bot that connects to Jellyfin and Jellyseerr. Configuration is done via an .env file in the project root.

Quick Setup Steps:

  1. Clone the repository from GitHub.
  2. Create a .env file by copying .env.example.
  3. Fill in your API keys and IDs (Discord, Jellyseerr, TMDB, etc.).
    Detailed .env configuration
    • DISCORD_TOKEN: Your bot's unique token. You can get this from the "Bot" section in the Discord Developer Portal.
    • BOT_ID: The Client ID of your bot. Find this on the "General Information" page of your Discord application.
    • GUILD_ID: The ID of the Discord server. Enable Developer Mode in Discord, then right-click your server icon and "Copy Server ID".
    • JELLYSEERR_URL: The full URL to your Jellyseerr API. Make sure to include /api/v1 at the end.
      • Node.js (without Docker): Use http://localhost:5055/api/v1 if Jellyseerr is on the same machine.
      • Docker: If Jellyseerr is on the host machine, use http://host.docker.internal:5055/api/v1.
    • JELLYSEERR_API_KEY: Your API key from Jellyseerr. Find it in Settings -> API Keys.
    • TMDB_API_KEY: Your API key from The Movie Database (TMDB).
    • OMDB_API_KEY: (Optional) Your API key from OMDb API for richer data.
    • JELLYFIN_BASE_URL: The publicly accessible URL of your Jellyfin server. This must be reachable from the internet for "Watch Now" links to work.
    • JELLYFIN_SERVER_ID: The unique ID of your Jellyfin server.
    • JELLYFIN_CHANNEL_ID: The ID of the Discord channel for new media notifications.
    • WEBHOOK_PORT: The network port the bot will listen on for Jellyfin webhooks (default is 8282).
  4. Run the application using docker compose up -d --build.
  5. In your Jellyfin settings, add the webhook URL:
    http://<your-host-ip>:<WEBHOOK_PORT>/jellyfin-webhook

For detailed instructions, troubleshooting, and configuration options, please refer to the project's official documentation.
